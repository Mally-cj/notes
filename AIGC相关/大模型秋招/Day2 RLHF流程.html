介绍下 RLHF 流程

大模型的 RLHF 过程：

  1. 预训练 +SFT 得到模型 A，作为 Policy model，参数为 θ
  2. 将模型 A 作为奖励模型 R（reward model）的初始值,用偏好数据来训练得到 R,参数为 ϕ
  3. 使用 PPO 算法优化模型 A。在优化过程中，除了要使用训练好的 R，还要使用另外两个模型，Reference Model 和 Value model（也叫 critic model）。

其中 PPO 算法中，还要

# PPO 算法损失函数的数学表达 结合 ToRL 代码，我来详细解释 PPO 算法的损失函数及其数学表达。 ## PPO 核心损失函数 PPO (Proximal Policy Optimization) 算法的核心是使用"裁剪"目标函数来限制策略更新步长。在 ToRL 项目中，这一目标函数在 core_algos.py 的 `compute_policy_loss` 函数中实现。 ### 1. 策略目标函数 (Policy Loss) PPO 的策略目标函数是： $$L^{CLIP}(\theta) = \hat{\mathbb{E}}_t \left[ \min \left( r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]

其中：

  • rt(θ)=πθ(at|st)πθold(at|st) 是新旧策略的概率比率
  • A^t 是优势估计
  • ϵ 是裁剪参数（在代码中是 cliprange）

在代码中的实现如下：

def compute_policy_loss(old_log_prob, log_prob, advantages, eos_mask, cliprange):
    # 计算概率比率 r_t(θ)
    negative_approx_kl = log_prob - old_log_prob
    ratio = torch.exp(negative_approx_kl)
    
    # 未裁剪的目标: -r_t(θ) * A_t
    pg_losses = -advantages * ratio
    # 裁剪后的目标: -clip(r_t(θ), 1-ε, 1+ε) * A_t
    pg_losses2 = -advantages * torch.clamp(ratio, 1.0 - cliprange, 1.0 + cliprange)
    
    # 取两者中较大者（实际是较小，因为有负号）
    pg_loss = verl_F.masked_mean(torch.max(pg_losses, pg_losses2), eos_mask)

裁剪机制确保了策略更新幅度不会过大，从而提高训练稳定性。

2. 熵损失 (Entropy Loss)

为了鼓励探索，PPO 常添加熵正则化项：

LENT(θ)=−βE^t[Hπθ]

其中：

  • Hπθ 是策略 πθ 在状态 st 下的熵
  • β 是熵系数（在代码中是 entropy_coeff）

在 ToRL 中，熵损失计算如下：

# 在 dp_actor.py 中计算熵损失
entropy, log_prob = self._forward_micro_batch(micro_batch=data, temperature=temperature)
# 应用掩码以排除工具输出部分
entropy_loss = verl_F.masked_mean(entropy, response_mask)

熵的计算在 torch_functional.py 中：

def entropy_from_logits(logits: torch.Tensor):
    """Calculate entropy from logits."""
    pd = torch.nn.functional.softmax(logits, dim=-1)
    entropy = torch.logsumexp(logits, dim=-1) - torch.sum(pd * logits, dim=-1)
    return entropy

这对应于熵的数学定义：H(p)=−∑ipilog⁡pi

3. KL 散度惩罚 (KL Penalty)

为了防止策略偏离参考模型过远，ToRL 还可选地添加 KL 散度惩罚项：

LKL(θ)=βKLE^t[DKL[πθref(⋅|st)||πθ(⋅|st)]]

其中 βKL 是 KL 散度系数（在代码中是 kl_loss_coef）。

在 ToRL 中实现如下：

# 在 dp_actor.py 中
if self.config.use_kl_loss:
    ref_log_prob = data['ref_log_prob']
    kld = core_algos.kl_penalty(logprob=log_prob, ref_logprob=ref_log_prob, kl_penalty=self.config.kl_loss_type)
    policy_loss = policy_loss + kld * self.config.kl_loss_coef

KL 散度的计算在 core_algos.py 中：

def kl_penalty(logprob: torch.FloatTensor, ref_logprob: torch.FloatTensor, kl_penalty) -> torch.FloatTensor:
    """KL divergence penalty between current policy and reference policy."""
    if kl_penalty == "low_var_kl":
        # 使用低方差估计的 KL 散度
        kld = (ref_logprob - logprob)
        return torch.clamp(kld, min=-10, max=10)

这是 KL 散度的一种特殊估计方式，旨在减少方差。完整的 KL 散度定义是：

DKL[p||q]=∑ipilog⁡piqi=∑ipi(log⁡pi−log⁡qi)

4. 总损失函数

将上述所有损失组合起来，ToRL 中的完整 PPO 损失函数为：

LTOTAL(θ)=LCLIP(θ)−βLENT(θ)+βKLLKL(θ)

在代码中实现为：

# 策略梯度损失（带裁剪）
pg_loss, pg_clipfrac, ppo_kl = core_algos.compute_policy_loss(old_log_prob, log_prob, advantages, eos_mask, clip_ratio)

# 熵损失
entropy_loss = verl_F.masked_mean(entropy, response_mask)

# 组合损失（负号是因为我们希望最大化熵）
policy_loss = pg_loss - entropy_loss * entropy_coeff

# 可选：KL 散度惩罚
if self.config.use_kl_loss:
    # ...添加 KL 散度惩罚
    policy_loss = policy_loss + kld * self.config.kl_loss_coef

5. 掩码机制 (Masking)

ToRL 项目的一个重要创新是 "Sandbox Output Masking"，这体现在损失计算的掩码应用上：

# 应用工具输出掩码到响应掩码
response_mask = attention_mask[:, -response_length:]
response_mask = response_mask & tool_output_masks

掩码应用到各种损失计算中，确保模型不会尝试记住沙盒环境的特定输出：

Lmasked=∑imaski⋅lossi∑imaski

在代码中通过 masked_mean 函数实现：

def masked_mean(values, mask, axis=None):
    """Compute mean of tensor with a masked values."""
    return (values * mask).sum(axis=axis) / mask.sum(axis=axis)

6. 优势估计 (Advantage Estimation)

ToRL 支持多种优势估计方法，包括 GRPO (Generalized Reward Penalized Objective)：

A^iGRPO=Ri−μiσi+ϵ

其中：

  • Ri 是轨迹 i 的总回报
  • μi 和 σi 分别是同一提示下所有响应的回报均值和标准差
  • ϵ 是小常数，防止除零

在代码中实现为：

def compute_grpo_outcome_advantage(token_level_rewards, eos_mask, index, epsilon=1e-6):
    # ...
    # 计算每组的均值和标准差
    for idx in id2score:
        id2mean[idx] = torch.mean(torch.tensor(id2score[idx]))
        id2std[idx] = torch.std(torch.tensor([id2score[idx]]))
        
    # 标准化每个响应的分数
    for i in range(bsz):
        scores[i] = (scores[i] - id2mean[index[i]]) / (id2std[index[i]] + epsilon)

总结

ToRL 项目中的 PPO 损失函数结合了多个组件：

  1. 策略裁剪损失 - 确保策略更新步长适中
  2. 熵正则化 - 鼓励探索
  3. KL 散度惩罚 - 防止策略偏离参考模型
  4. 掩码机制 - 防止模型记忆沙盒输出
  5. 优势估计 - 针对特定任务的回报归一化

这些损失函数的组合和掩码机制的应用，使 ToRL 能够有效地训练语言模型进行工具集成的强化学习，同时保持训练的稳定性和防止过度记忆特定输出。
